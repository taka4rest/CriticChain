SUMMARY: Agent Collaboration Pattern
================================================================================

Processing Flow Pattern:
  1. Lint Check      → 構文・構造の検証（形式的エラーを検出）
  2. Research        → 外部知識の収集（関連トピックの検索）
  3. Analysis        → シニアAIエンジニアによる評価
  4. Fact Check      → 分析内容の正確性検証（ハルシネーションチェック）
  5. Draft Review    → レビュー下書き作成
  6. Critique        → 品質管理（レビュー内容の批評）
  7. Refine Review   → 改善イテレーション（複数回）

Agent Collaboration Dynamics:

  🔍 Linter Agent
     → 形式的エラーを検出（区切り文字、変数定義など）
     → 改善案を提示

  👨‍💻 Senior Engineer Agent
     → 技術的観点から評価
     → Chain of Thought、堅牢性などを分析
     → S/A/B/Cランクで判定

  🕵️ Fact Checker Agent
     → エンジニアの評価が正確かチェック
     → ハルシネーション検出
     → コンテキスト断絶の検証

  ✍️ Review Writer Agent
     → 学習者向けレビューを作成
     → 「良かった点」「改善点」を整理

  🎯 Critic Agent
     → レビュー内容を批評
     → 判定の甘さ、トーンの問題を指摘
     → より厳格な修正を要求

  🔄 Refiner Agent
     → Criticのフィードバックを反映
     → 複数回のイテレーションで品質向上

Key Observations:
  - セッション1: 3回のRefineイテレーションで「合格」→「条件付き合格」に厳格化
  - セッション2: 同じプロンプトで一貫した評価（システムの再現性を確認）
  - セッション3: 改善後のプロンプトをPASSと判定（改善の効果を検証）
  - エージェント間の相互チェックにより、評価の質と一貫性を担保
